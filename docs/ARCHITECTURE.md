# Architecture (FSD)

## Слои и зависимости
- **Слои:** app → screens → widgets → features → entities → shared. Не поднимаем код вверх по слоям; зависимости только “вниз”.
- **Public API:** каждый срез экспортирует наружу только через `index.ts` в корне слоя/среза.
- **Внутренняя структура среза:** `ui/`, `model/` (типизация, сторы, сервисы), `lib/` (утилиты), `api/` (запросы), `config/` (константы), `styles/` при необходимости. Типы всегда в `model/*.types.ts` (camelCase).

## UI‑компоненты
- **Без запросов:** сущности/виджеты/экраны не делают API-вызовы внутри UI; вся бизнес-логика прокидывается пропсами сверху (например, `onSubmit` для форм).
- **Отдельные стили:** styled-components в `*.styles.ts` рядом с компонентом. Для SSR используем провайдер styled-components, чтобы избежать FOUC.
- **Тексты:** пользовательские тексты на русском, ошибки валидации тоже.

## Формы и валидация
- Formik + Yup для форм; валидация в `model` (схемы), UI только показывает ошибки.
- Сетевые запросы (например, `signIn`) выполняются на уровне экрана/страницы, результат и ошибки передаются в UI пропсами.

## Провайдеры и инфраструктура
- Общие провайдеры (Ant Design, styled-components, React Query и т.п.) подключаем в `app/(providers)/WithProviders`.
- Авторизация: proxy (middleware) с `auth()`; публичные маршруты и константы путей лежат в `shared/config/routes`. Неавторизованных редиректим на `/sign-in` c `callbackUrl`.

## Именование и стили
- Файлы типов: camelCase `*.types.ts`, `*.styles.ts`.
- Стили и компоненты — ASCII, без лишнего шума; один компонент — один файл.
