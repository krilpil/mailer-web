# Product Context

## Problem Statement
- Нужен веб-продукт уровня MWP, который закрывает базовые сценарии email-сервиса: управление доменами, почтовыми ящиками, шаблонами, подписчиками и отправкой писем.
- Текущий mail-backend строится вокруг BillionMail, но ему нужна отдельная удобная веб-панель и бизнес-логика со стороны продукта.
- Пользователю нужен единый интерфейс, где можно не только работать с рассылками, но и управлять доменной инфраструктурой, DNS-статусами и отправкой писем через API.
- Система должна упростить запуск собственного email-продукта без необходимости вручную работать с почтовым сервером, Swagger и низкоуровневыми настройками BillionMail.

## Target Users
- Администраторы и операторы веб-панели Mailfinch Core.
- Команды, которым нужен self-hosted email-продукт с управлением доменами и почтовыми ящиками.
- Пользователи, которые отправляют транзакционные письма и простые маркетинговые рассылки через веб-интерфейс и API.

## User Needs
- Регистрация, логин и восстановление пароля.
- Создание и управление доменами, проверка DNS-записей, SPF, DKIM, DMARC и PTR.
- Создание и управление почтовыми ящиками для доменов.
- Работа со списками подписчиков, статусами подписки, импортом и экспортом CSV.
- Создание, хранение и переиспользование шаблонов писем.
- Редактирование HTML-шаблонов с переменными.
- Создание кампаний: черновик, выбор шаблона, выбор аудитории, отправка сразу или по расписанию.
- Поддержка unsubscribe-сценария.
- Отправка писем через API от имени конкретного почтового адреса.
- Понятный статус ошибок и стабильная работа интеграции с BillionMail API.

## Value Proposition
- Единый интерфейс для основных почтовых операций: домены, ящики, шаблоны, подписчики и отправка писем.
- Быстрый запуск собственного email-продукта на базе BillionMail без необходимости постоянно работать напрямую через его панель и внутренние API.
- Упрощение доменной и почтовой настройки за счет отображения DNS-статусов и централизованного управления.
- Возможность связать продуктовую логику на Next.js с внешним mail-backend и дать пользователю более удобный UX.

## Success Metrics
- Пользователь может успешно пройти сценарии sign-up, sign-in и password reset.
- Пользователь может создать домен, проверить DNS-статус и создать нужный почтовый ящик.
- Пользователь может создать шаблон, кампанию и отправить письмо без ручной работы в BillionMail.
- Успешно работает сценарий API-отправки писем от нужного адреса.
- Низкая доля ошибок в операциях создания доменов, ящиков и отправки писем.
- Снижение количества ручных действий в BillionMail для типового сценария пользователя.
